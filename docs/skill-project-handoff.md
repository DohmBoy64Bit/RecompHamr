# project-handoff

Project documentation skill. Load when you need to generate or update project
handoff docs, state summaries, or onboarding materials.

## What it teaches

- Generate structured project documentation for handoff or resume
- Include: goal, source of truth, toolchain, current phase, blockers,
  learned patterns, active commands
- Write for the NEXT person (or your future self) — not a log, a map
- Save to `.rehamr/` files that `/status-re` can surface

## When to use

At major milestones, before handing off to another developer, or before
stepping away from a multi-session project. Pairs naturally with the
persistent memory system (`REPHAMR_STATE.md`).

## Audit dispositions

| Legacy contract | Disposition | Current evidence |
|---|---|---|
| One turn cancellation root, ordered calls, complete pairing, stale close/result rejection, final accounting | improved | `internal/agent` uses stable turn/round/work identities and opaque work rather than mutable TUI channel/context identity. Running-tool cancellation additionally removes the incomplete goal to prevent side-effect reissue. |
| Same-target failure, runaway, empty-reply, leaked-call, and verification policies | equivalent | Thresholds, queue-drain ordering, re-arm/latch/reset behavior, honest `unverified` suppression, and soft-nudge semantics match. Current branding and the already-accepted verification wording are non-compatibility text refinements. |
| Budgeted newest-first packing, head/tail truncation, tool pairing, task anchoring, system-nudge demotion | improved | Current positional pairing rejects reused-ID cross-turn false matches and recovers the newest complete tool group past trailing nudges or empty replies without resurrecting partial groups. |
| Reasoning/content/tool-fragment SSE, reasoning fallback, probe, cancellation, activity watchdog | improved | Current transport adds mid-stream error-frame detection and bounded transient pre-stream retries behind injected timing/HTTP boundaries. |
| Five-minute Legacy idle default | intentionally changed | Current accepted default is one hour, still activity-reset and environment-overridable, to avoid killing slow local inference during long prefill/generation. |
| Hosted budget/quota headers and 402 account behavior | not applicable | Intentionally removed and accepted in Stage A; generic authentication, status errors, and context-window probing remain. |
| Skill-template classifier | deferred | Its inputs, outputs, tests, and only user integration are skills and skill slash commands; Stage G owns the standards-based replacement decision. |

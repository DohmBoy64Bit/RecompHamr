# `recomp-foundations` Legacy Skill Disposition

Legacy `RecompHamr-Legacy-main/internal/skills/recomp-foundations.md` was read completely. It is a large static URL and curriculum-path catalog rather than an executable workflow or focused expertise package, and it has no directly linked local resources. The mandatory current Agent Skills authority set and official Codex skill guidance had been read before this individual disposition.

## Decision

**Intentionally changed — not migrated as a skill.** Its useful behavior is current-source discovery and source-quality judgment. RecompHamr already provides bounded `recomp_reference` and `repomixr` capabilities, while current web/reference lookup must verify URLs, versions, command syntax, and source authority at use time. Embedding the Legacy list would duplicate dozens of mutable external claims, consume catalog/maintenance budget, and create false confidence when links or project layouts change.

Platform skills may name a small number of essential upstream authorities only when individually verified during their migration and clearly label external references as learning aids rather than evidence about the user's target. This disposition does not claim that every Legacy URL is currently valid; no bulk link-validity claim was made.

No `SKILL.md`, script, resource, or eval set is produced because a non-migrated static directory should not trigger. The Legacy file remains traceable here, and its individual disposition is final for Stage G unless the user approves a separately maintained, current reference-index product.

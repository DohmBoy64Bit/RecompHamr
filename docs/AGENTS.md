# Documentation Subtree Instructions

These rules add to the root `AGENTS.md` for work under `docs/`.

- Treat current source and tests as stronger evidence than historical prose.
- Do not update a claim merely to make a documentation contract keyword pass; verify the underlying behavior.
- Keep user docs task-oriented and developer docs evidence-oriented.
- Update `docs/README.md` when adding, removing, or relocating durable documentation.
- Update `docs/documentation-contract.json` when a new durable document or mandatory contract term must be enforced.
- Keep relative links valid.
- Distinguish `verified`, `unverified`, `blocked`, and `unsupported` claims.
- Do not copy external documentation wholesale. Summarize requirements and link to the current authoritative source.
- When documenting Legacy parity, record the required behavior and chosen disposition rather than implying source-level identity.
- When documenting Agent Skills work, use `docs/dev/roadmap/agent-skills-standard.md` as the repository policy and verify current external authority before changing normative claims.
